# Design System

A token-driven design system with a **light/dark theme generated by symmetry**, plus component specs and a documentation site. Built to be consumed by both developers and AI tools.

## What's inside
```
tokens/        Primitive palette + semantic light/dark themes (+ THEMES.md rule)
components/    Component specs, each referencing semantic tokens only
dist/          tokens.css — compiled CSS custom properties (light + dark)
docs/          Documentation website (GitHub Pages)
scripts/       Regenerate the dark theme & CSS after editing tokens
CLAUDE.md      Rules for AI tools using the system
llms.txt       (in docs/) machine index
```

## Theming
Components reference **semantic tokens** (e.g. `--accent-primary`, `--text-default`). Light values live in `:root`; dark values under `[data-theme="dark"]`. Dark is derived from light by mirroring the palette: neutrals `105 − step` (+ `white↔black`), chromatic `110 − step`. The accent ramp shifts hue, so the brand is blue in light and cyan in dark. See `tokens/THEMES.md`.

## Use the CSS
```html
<link rel="stylesheet" href="dist/tokens.css">
<!-- light by default; add data-theme="dark" on <html> for dark -->
```

## Documentation site (GitHub Pages)
The `docs/` folder is a static site. To publish: repo **Settings → Pages → Source: Deploy from a branch → main → /docs**. It will be served at `https://<you>.github.io/<repo>/`.

## Regenerate after edits
```bash
python3 scripts/build-dark.py   # rebuild semantic-dark.json from semantic-light.json
python3 scripts/build-css.py    # rebuild dist/tokens.css
```
